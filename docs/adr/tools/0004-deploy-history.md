# ADR 0004: 배포 이력 조회 — CloudTrail 기반 배포/변경 이벤트 조회

Date: 2026-04-21

## Status

Proposed

## Context

장애와 최근 배포의 상관관계를 분석하려면 배포/변경 이력을 자동으로 조회해야 한다. 단순히 시간 범위로 컷오프하면 느린 누수(커넥션 릭, 메모리 릭)처럼 배포 후 수시간~수일 후에 발현되는 장애 원인을 놓칠 수 있다.

## Decision

**CloudTrail LookupEvents 기반 MCP 도구 + 직전 N개 이벤트 조회** 전략을 채택한다.

### 핵심 결정사항

1. **MCP 도구 인터페이스**: `query_deploy_history(service_name, limit_count)` 형태의 도구를 제공한다. CloudTrail LookupEvents API를 래핑한다.

2. **직전 N개 조회**: 알람 대상 서비스의 직전 N개(기본 5개) 배포 이벤트를 시간 제한 없이 조회한다. 24시간 이상 전 배포도 점진적 장애의 원인일 수 있으므로 시간 기반 컷오프를 두지 않는다.

3. **조회 대상 이벤트**: CodeDeploy 배포, ECS 태스크 정의 변경, Lambda 함수 업데이트, CloudFormation 스택 변경 등 인프라/어플리케이션 변경 이벤트를 대상으로 한다.

4. **상관관계 분석**: 각 배포 시각과 메트릭 이상 시작 시각의 시간차를 계산하여 가장 상관성 높은 배포를 식별한다.

5. **CloudTrail 지연 반영**: CloudTrail 이벤트가 최대 15분 지연될 수 있으므로 조회 시간 범위에 여유를 반영한다.

## Consequences

### Positive

- 시간 기반 컷오프 없이 직전 N개 배포를 조회하여 느린 누수 장애의 원인도 식별 가능
- 배포 시점과 메트릭 이상 시점의 자동 상관관계 분석으로 의심 배포 즉시 식별

### Negative

- 직전 N개 배포가 매우 오래전인 경우 무관한 배포까지 조회하여 노이즈 증가 가능
- CloudTrail 이벤트 지연으로 가장 최근 배포가 누락될 수 있음

### Risks

- 조회 기간 내 배포 이벤트가 없는 경우 "배포 없음"으로 기록하여 배포 관련 가설 기각 근거로 활용한다.
